## Java代码注释规范化处理提示词

### 核心任务目标

为Java项目添加或更新符合项目规范的中文注释，确保代码可读性和维护性。

### 注释格式规范

#### 1. 类注释规范

根据情况来处理类注释, 如果是特别复杂且重要的类, 需要按照下面的规范来生成类注释, 如果只是一个简单的类, 一句话描述清楚即可.

```java
/**
 * 类功能描述，详细说明类的用途、设计模式、主要特性等
 * <p>
 * 详细描述段落，说明类的核心功能、使用场景、技术特点
 * 支持多个<p>段落来组织复杂的描述内容
 * <p>
 * 主要特性：
 * - 特性1描述
 * - 特性2描述  
 * - 特性3描述
 * <p>
 * 使用示例（如需要）：
 * {@code 示例代码}
 * <p>
 * 设计模式/架构说明（如适用）
 *
 * @author dong4j
 * @version 1.0.0
 * @email "mailto:dong4j@gmail.com"
 * @date {yyyyyy.MM.dd HH:mm}
 * @since 1.0.0
 */
```

#### 2. 方法注释规范

同样的, 如果是非常重要的方法, 需要按照下面的规范来生成方法注释, 如果只是一个简单的方法, 一句话描述即可, 但是要写清楚出参入参和返回结果,
如果存在异常也要添加。方法注释统一添加 @since 1.0.0。

```java
/**
 * 方法功能描述，说明方法的作用和执行逻辑
 * <p>
 * 详细描述方法的执行流程、业务逻辑、注意事项等
 * 包括方法的使用场景和限制条件
 *
 * @param paramName 参数描述，说明参数的作用、类型要求、取值范围等
 * @param anotherParam 另一个参数的详细描述
 * @return 返回值描述，说明返回内容的含义、格式、可能的值等
 * @throws ExceptionType 异常描述，说明什么情况下会抛出此异常
 * @throws AnotherException 其他可能抛出的异常及其条件
 * @since 1.0.0
 */
```

#### 3. 字段注释规范

java

```java
/** 字段功能描述，说明字段的用途、默认值、生命周期等 */
private Type fieldName;

/** 
 * 复杂字段的详细描述
 * <p>
 * 包含使用场景、注意事项、相关配置等信息
 */
private ComplexType complexField;
```

### 关键规则要求

#### 格式要求

1. 禁止使用单独占据一行的 `\*`，必须使用`* <p>`格式
2. **所有注释必须使用中文**，如遇英文注释需翻译并改写
3. **作者信息统一为dong4j**，邮箱为dong4j@gmail.com
4. **日期统一使用 yyyy.MM.dd hh.mm 格式**
5. **版本号统一为1.0.0**
6. **中英文之间添加空格**

#### 内容要求

1. **功能描述要详细准确**，避免空泛的描述
2. **技术术语要准确**，体现专业性
3. **包含使用场景和注意事项**
4. **对于复杂类要说明设计模式和架构**
5. **参数和返回值描述要具体**
6. **异常情况要完整说明**

#### 代码行注释

1. **关键业务逻辑处添加中文注释**
2. **复杂算法要分步骤注释**
3. **重要配置项和常量要说明用途**
4. **临时解决方案要标注TODO或FIXME**

### 处理流程规范

#### 1. 文件处理顺序

- 按模块优先级处理（核心模块优先）
- 先处理接口和抽象类
- 再处理具体实现类
- 最后处理工具类和常量类

#### 2. 错误处理策略

- 遇到匹配失败时，重新读取文件内容确保准确性
- 对于复杂文件，可以分段处理避免单次操作过大
- 使用edit*file作为search*replace的备用方案
- 处理完成后必须使用get_problems验证语法正确性

#### 3. 质量检查要点

- 确保所有public类和方法都有注释
- 检查注释格式符合规范要求
- 验证中文描述准确性和专业性
- 确保没有遗漏重要的业务逻辑说明

### 特殊情况处理

#### 1. 框架特定注解

- 对于Spring相关注解，要说明其在框架中的作用
- SPI相关注解要解释服务发现机制
- 自定义注解要详细说明使用方法和效果

#### 2. 设计模式识别

- 工厂模式：说明对象创建策略
- 策略模式：说明不同策略的适用场景
- 观察者模式：说明事件处理机制
- 代理模式：说明代理的目标和方式

### 输出质量标准

#### 必须达到的标准

1. **所有注释使用规范的中文表达**
2. **技术描述准确专业**
3. **格式完全符合项目规范**
4. **没有语法错误或编译问题**
5. **注释内容与代码逻辑一致**
6. **中英文之间添加空格**

#### 评判优秀注释的标准

1. **让不熟悉代码的开发者能快速理解**
2. **包含足够的上下文信息**
3. **体现代码的设计思想**
4. **包含实际的使用指导**
5. **对潜在问题有预警说明**

### 使用示例

处理时严格按照以上规范执行，确保每个Java文件的注释都能达到企业级代码的文档标准，为后续的维护和开发提供完善的文档支持。
